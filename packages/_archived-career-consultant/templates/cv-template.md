# CV Template - Format Guide

**File Name**: `cv-{focus-area}.md` (e.g., `cv-backend.md`, `cv-general.md`)  
**Location**: `user-data/profile/cvs/`  
**Format**: Markdown (plain text)

---

## Instructions

This template shows the recommended format for your CV files. The skill will:
- Extract skills and experience from your CV
- Match job requirements against your background
- Calculate match scores
- Identify skill gaps

**Tips**:
- Use clear section headers
- List specific technologies and tools
- Include quantifiable achievements
- Keep each CV focused on a specific career direction
- Update regularly with new skills and projects

## Notes for Skill Usage

**How the skill uses this CV**:
1. Extracts technical skills and experience
2. Matches against job requirements
3. Calculates match score (0-35 points)
4. Identifies skill gaps (critical vs. nice-to-have)
5. Determines which CV variant is best match

**Optimization Tips**:
- Use specific technology names (not "various tools")
- Include years of experience with key technologies
- Add quantifiable achievements with metrics
- Keep technical skills section comprehensive
- Update after completing major projects

**Multiple CV Strategy**:
If you maintain multiple CV variants, focus each on:
- **Backend CV**: Heavy on server-side technologies, databases, APIs
- **Frontend CV**: Heavy on UI frameworks, JavaScript, UX
- **Full-Stack CV**: Balanced coverage of both
- **Management CV**: Leadership, team building, strategy
- **Specialized CV**: Focus on specific domain (ML, DevOps, Security, etc.)
